WSL/Homebrew Setup

I- Windows Subsystem for Linux (WSL) setup

WSL 是 微软在Win10中推出的一个新功能（需要版本升级到1709以上，推荐1903以上，因为可以安装Windows Terminal）。通过WSL，可以在Windows下，不开虚拟机的情况下使用一个linux内核（非完整）。

NB：本教程将安装WSL1。Windows 10 2004 版（Windows update推动中，可能暂时不完全稳定）中推出了新的WSL2，可以使用完整的Linux内核，但是访问windows 下的文件速度降低，应按需升级。

1- 安装WSL

  • 开启WSL功能
    • 前往 控制面板-程序和功能-启用或关闭Windows功能
    • 启用“适用于Linux的Windows子系统” 功能并点击确定
    • 重启电脑
  • 安装Linux 发行版
    • 到Microsoft Store里下载想要安装的Linux Distribution (此处推荐Ubuntu 18.04 (或者20.04))
    • 安装好了后启动（首次启动会花费一点时间），然后设置用户名/密码
  • 更新Apt包信息与升级
    sudo apt-get update
sudo apt-get upgrade
  • OPTIONAL: 换国内源（此处使用清华源）
    • 先备份原文件
      sudo cp /etc/apt/sources.list /etc/apt/sources.list.bak
    • 然后用喜欢的文本编辑器打开/etc/apt/sources.list

      例：

      sudo nano /etc/apt/sources.list
#或者
sudo vim /etc/apt/sources.list
    • 将原文件中的东西全部删除或者注释掉（在每行前面打#），然后插入：
      deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal main restricted
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-updates main restricted
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal universe
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-updates universe
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal multiverse
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-updates multiverse
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-backports main restricted universe multiverse
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-security main restricted
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-security universe
deb http://mirrors.tuna.tsinghua.edu.cn/ubuntu/ focal-security multiverse
    • 然后重复第三步（更新Apt包信息与升级）
  • 安装完毕后，就可以用以下命令来安装需要的包了
    sudo apt-get install [the package you want]

NOTE: 20.04版本中还有些奇怪的Bug (18.04没有的)... 例如安装NodeJS 12的时候会出现：

curl -sL https://deb.nodesource.com/setup_12.x | sudo -E bash -
#错误:
gpg: can't connect to the agent: IPC connect call failed

这个我是用如下方法解决的：

sudo apt remove gpg
sudo apt-get update -y
sudo apt-get install -y gnupg1

然后先Curl掉需要gpg包的东西（安装NodeJS的时候会自动重新安装有bug的gpg包）

curl -s https://deb.nodesource.com/gpgkey/nodesource.gpg.key | sudo apt-key add -

然后再根据官网：

# Using Ubuntu
curl -sL https://deb.nodesource.com/setup_12.x | sudo -E bash -
sudo apt-get install -y nodejs

（这个错误在安装yarn的时候也有，可以同样的方式remove gpg之后先curl，再安装）

2- 在VSCode中使用WSL

有以下两个选择：

A- 在Windows下，将WSL当Terminal用

  • VSCode中打开Terminal
  • 点击右上角的下拉框 点击Select Default shell
  • 选择WSL bash
  • 打开新Terminal便可以看到你的WSL了。

NOTE: 通过WSL访问Windows文件时，应访问 /mnt/[你的盘符, 比如c或者d]/你的文件路径

例如我这里就是 /mnt/c/Users/[...]

B- 用Remote WSL功能 在WSL下启用VSCode

  • 安装Remote WSL 插件
  • 通过以下两种方式之一启动：
    • 按F1然后写 Remote-WSL : New Window ...

    或者

    • 点击左边栏的这个地方打开Remote explorer

    然后：

    • 选择想要的Distro并点击右边的小按钮
  • 然后就可以启动了 (启动时要先等它Configure好，会需要一段时间)

II- Homebrew setup (Mac)

安装：

拷贝以下代码至terminal

/bin/bash -c "$(curl -fsSL [https://raw.githubusercontent.com/Homebrew/install/master/install.sh](https://raw.githubusercontent.com/Homebrew/install/master/install.sh))"

使用：

brew update #更新目录
brew upgrade #更新所有安装的软件包
brew list #列表所有安装的软件包
brew install {package name} #安装对应的软件包

Homebrew是一款针对macOS的软件包管理工具，具有安装、卸载、更新、检索等功能来帮助Mac用户简便地实现包管理。Homebrew分为git repo和bottles两部分。安装Homebrew时，默认源为brew.git，因此在国内访问速度较慢。通过简单的几个步骤，我们可以换成国内的镜像来解决这一问题。

一. 中科大镜像源

  1. 更换默认源
    # 更换brew.git
1 | cd "$(brew --repo)"
2 | git remote set-url origin https://mirrors.ustc.edu.cn/brew.git
    # 更换homebrew-core.git
1 | cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
2 | git remote set-url origin https://mirrors.ustc.edu.cn/homebrew-core.git
3 | cd
4 | brew update

  1. 更换Homebrew bottles源
    1 | echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.ustc.edu.cn/homebrew-bottles' >> ~/.bash_profile
2 | source ~/.bash_profile

二. 清华镜像源

  1. 更换默认源
    # 更换现有上游
1 | cd "$(brew --repo)"
2 | git remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/brew.git
3 | cd "$(brew --repo)/Library/Taps/homebrew/homebrew-core"
4 | git remote set-url origin https://mirrors.tuna.tsinghua.edu.cn/git/homebrew/homebrew-core.git
5 | cd
6 | brew update

2. 更换Homebrew bottles源

1 | echo 'export HOMEBREW_BOTTLE_DOMAIN=https://mirrors.tuna.tsinghua.edu.cn/homebrew-bottles' >> ~/.bash_profile
2 | source ~/.bash_profile
